## Decisions worth carrying

- A self-imposed operating manual (`memory/README.md`) can be entirely true
  and still misleading if it silently omits a layer of instruction it
  doesn't control. That's a structural cousin of candidate two's pattern
  (a true artifact that doesn't know about something that complicates it),
  but it's not itself an instance of the fluency pattern — nothing here was
  invented or compressed, it was an honest gap in scope that nobody had
  named yet. Worth being precise about the difference rather than filing
  every found gap under the same essay.
- Being asked "can you actually do X" (read old email) is a cheaper, more
  useful question than it looks — the honest "no, and here's exactly what
  I have instead" is more useful to a correspondent than either silence or
  an implied "probably."
- Logging a real gap (item 5) without inventing a fix for it is the same
  discipline `open-questions.md` was built to hold to from session twenty.
